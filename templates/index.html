Ejemplo de conversión de CSV a SQL

Supongamos que tienes un archivo plano en formato CSV con información de usuarios. Este tipo de archivos es muy común cuando se exportan datos desde Excel, sistemas administrativos o aplicaciones web. Un archivo CSV almacena la información en columnas separadas por un delimitador, normalmente una coma ,.

Por ejemplo, si tienes el siguiente archivo llamado usuarios.csv: 

id,nombre,edad
1,Ana,25
2,Carlos,30
3,María,22

En este caso el archivo contiene tres columnas: id, nombre y edad. La primera fila corresponde a los nombres de las columnas y las siguientes filas representan los registros que queremos insertar en una base de datos.

Cuando subes este archivo al generador SQL, la herramienta realiza automáticamente varios pasos:

  • Detecta el delimitador del archivo (coma, punto y coma, tabulación, etc.).
  • Analiza los nombres de las columnas.
  • Determina el tipo de dato de cada columna (texto, número, fecha, etc.).
  • Genera un script SQL compatible con la base de datos seleccionada.

Script CREATE TABLE generado

Si eliges generar un script de tipo CREATE TABLE, la herramienta crea automáticamente la estructura de la tabla según las columnas detectadas en el archivo CSV. 

CREATE TABLE usuarios (
  id INT,
  nombre VARCHAR(50),
  edad INT
);

Este script permite crear la tabla en la base de datos antes de insertar los datos. Los tipos de datos se infieren automáticamente analizando el contenido de cada columna del archivo.

Script INSERT INTO generado

Si eliges generar un script de tipo INSERT INTO, la herramienta crea las instrucciones necesarias para insertar cada registro del archivo en la tabla. 

INSERT INTO usuarios (id,nombre,edad)
VALUES (1,'Ana',25);

INSERT INTO usuarios (id,nombre,edad)
VALUES (2,'Carlos',30);

INSERT INTO usuarios (id,nombre,edad)
VALUES (3,'María',22);

De esta forma puedes importar fácilmente grandes volúmenes de datos en tu base de datos sin tener que escribir manualmente cada sentencia SQL. Este tipo de conversión es muy útil cuando trabajas con bases de datos como SQL Server, PostgreSQL, MySQL u Oracle.
